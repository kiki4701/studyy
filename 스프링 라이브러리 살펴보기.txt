Gradle 이나 Maven 같은 이번 빌드 툴들은 요즘에 의존관계를 다 관리를 해준다 예를 들어 우리가 spring web 이 필요해서 선택을 하면 spring web 이 의존관계에 있는 것들을 땡겨오고 하면서 필요한 것들을 알아서 다 땡겨온다

예전에는 WAS라고 해서 웹서버(tomcat같은거)라는걸 직접 서버에 설치를 해놓은 후 거기에 자바 코드를 밀어넣는 식으로 웹서버랑 개발 라이브러리랑 분리가 되어있었다

요새는 소스 라이브버리 내에서 웹서버(tomcat)같은거 를 그냥 들고있는다 이런것을 임배디드를 내장하고 있다고 한다

이걸 통해서 자바 메인 메서드만 실행해도 바로 웹 서버가 뜨고 8080포트로 들어가 진다

slf4j는 쉽게 말해서 인터페이스이다 실제 로그를 어떤 구현체로 출력할거냐로 요새는 logback을 많이 선택한다 그래서 스프링부트를 떙기면 자동으로 slf4j 랑 logback 라이브러리가 스트링부트 스타터 로깅에 포함되어서 자동으로 땡겨진다

junit 은 테스트를 할 때 쓰는 라이브러리 mockito, assertj 이런것들은 테스트를 편리하게 하도록 도와주는 라이브러리이고 핵심은 junit (오랫동안 4버전 쓰다가 요새 5로 넘어가는 추세)

spring-test는 통합해서 테스트 할 수 있도록 도와주는 라이브러리
