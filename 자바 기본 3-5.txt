자바 기본 3-5

객체 지향 프로그래밍에서는 데이터와 기능을 하나로 묶어서 음악플레이어라는 개념을 온전히 하나의 클래스에 담는다.

개념을 객체로 온전히 만드는것이 더 중요하기 때문에 음악 플레이어가 어떤 속성(데이터)를 가지고 어떤 기능(메서드)을 제공하는지 이 부분에 초점을 맞춰야 한다.

MusicPlayer 클래스를 모두 정의 하고 난 후 이제 음악 플레이어가 필요한 곳에서 이것의 객체를 생성 한 후 필요한 기능(메서드)를 호출하기만 하면 된다.

필요한 모든 것은 MusicPlayer 안에 들어있기 때문에 사용하는 입장에서는 데이터인 volume, isOn 같은 데이터를 전혀 사용하지 않고 내부에 어떤 속성(데이터)이 있는지 전혀 몰라도 된다. 단순하게 MusicPlayer가 제공하는 기능 중에 필요한 기능을 호출해서 사용하시만 하면 된다.

 

캡슐화

MusicPlayer 를 보면 음악 플레이어를 구성하기 위한 속성과 기능이 마치 하나의 캡슐에 쌓여있는 것 같다. 이렇게 속성과 기능을 하나로 묶어서 필요한 기능을 메서드를 통해 외부에 제공하는 것을 캡슐화라 한다.

이러한 경우 내부 코드가 변하는 경우에 다른 코드는 변경하지 않아도 된다.

ex) 변수명이 변경 될 경우, 출력하는 메세지를 변경 할 경우

 

물론 외부에서 호출하는 메서드의 이름이 변경된다면 사용하는 곳의 코드도 변경해야 한다!!
