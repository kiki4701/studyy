JDK, JRE, JVM

JDK란
Java Development Kit 의 약자로 개발에 필요한 자바 컴파일러 같은 것들과 JRE, JVM 을 포함하고 있다. JDK를 설치하면 자동으로 다 설치되는 것이다. 컴파일러는 우리가 작성한 자바 문법을 컴퓨터가 이해할 수 있게 바꿔주는 해석기 같은 존재이다. 

JRE란
Java Runtime Environment 의 약자로 자바의 런타임, 개발에 유용한 자바 라이브러리와 자바 가상 머신(JVM), 자바 클래스 로더(Java class loader)를 포함하고 있다. 자바 프로그램을 실행만 한다면 JRE만 있어도 된다. 클래스 로더, 클래스 라이브러리를 통해 작성한 자바 코드를 라이브러리와 결합한 후 JVM에 넘겨 실행시킨다. JRE는 그 자체로 특별한 기능을 한다기보다는 JVM이 원활하게 잘 작동할 수 있도록 환경을 맞춰주는 역할을 한다. 

JVM란
Java Virtual Machine 의 약자로 자바 바이트 코드를 실행한다. 
2가지 특징이 있다.
1. 자바 프로그램이 어느기기, 어느 운영체제 상에서도 실행될 수 있게 만들어준다. => WORA
2. 자바 프로그램의 메모리를 효율적으로 관리 & 최적화해준다.

각 OS에 맞는 JVM을 설치하기만 하면 어떤 OS에서 작성한 자바코드던지 상관없이 JVM을 통해서 실행 할 수 있다. 

** 가비지 컬렉션(Garbage Collection)
JVM이 메모리를 관리하는 프로세스를 지칭하는 용어이다. 자바 프로그램 상에서 사용하지 않는 메모리를 지속적으로 찾아 제거함으로써 효율적인 메모리 관리를 가능하게 한다.
